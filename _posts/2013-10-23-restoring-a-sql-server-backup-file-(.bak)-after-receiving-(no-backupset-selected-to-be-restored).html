--- layout: post title: Restoring a SQL Server Backup File (.bak) After Receiving (No Backupset Selected to Be Restored) ---

Today, I ran into an issue where I need to restore a database file that I had generated on one of our remote servers. I had placed this file in my dropbox account and downloaded to my local machine.

Once on my machine, I moved the file to the MSSQL Backups folder and proceeded to try to restore the file. I was unable to do so and received this error, "No Backupset Selected to Be Restored". I viewed the permissions on the .bak file in question and another .bak file on my machine and noticed that the file I was unable to import was missing some permissions. With some help from this article, I was able to get the correct permissions on my file and restore the backup.

I had to add the following users to my .bak file via the "Properties/Security" tab for the file. These are the users to add:

  1. NT SERVICE\MSSQL$SQLEXPRESS (Give full permissions)
  2. SQLServerMSSQLUser$[Your-Computer-Name]$SQLEXPRESS (Give full permissions)
