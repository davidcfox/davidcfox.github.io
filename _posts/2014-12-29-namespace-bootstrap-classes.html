--- layout: post title: Namespace Bootstrap Classes ---

I have recently completed some work where I needed to namespace the Bootstrap classes to prevent interference with an existing skin on a site. The process for compiling LESS and SASS classes is essentially the same:

 .twb {
  @import 
   "bootstrap files";
 }

After some trial and error, I found it is advantageous to leave certain files out of the namespaced import. Several files, like the javascript components place their markup directly in the "body" tag, so unless you are applying your custom class to the page body, your styles will not be applied.

Here is the combination that worked for me in a DNN module:

@import
  // Some compass goodness
  "compass",
  "bootstrap/normalize",
  // Variables - override any bootstrap variables in here
 "variables",

// Core variables and mixins
"bootstrap-compass",
"bootstrap/variables",
"bootstrap/mixins",
// Components w/ JavaScript
"bootstrap/modals",
"bootstrap/tooltip",
"bootstrap/popovers",
"bootstrap/carousel",
"bootstrap/close";

.twb {
   @import

   // Place mixins. Override Bootstrap mixins you may wish
   "mixins",
   "sass_helpers/mixins",

   // Reset and dependencies
   "bootstrap/print",
   "bootstrap/glyphicons",

   // Core CSS
   "bootstrap/scaffolding",
   "bootstrap/type",
   "bootstrap/code",
   "bootstrap/grid",
   "bootstrap/tables",
   "bootstrap/forms",
   "bootstrap/buttons",

   // Components
   "bootstrap/component-animations",
   "bootstrap/dropdowns",
   "bootstrap/button-groups",
   "bootstrap/input-groups",
   "bootstrap/navs",
   "bootstrap/navbar",
   "bootstrap/breadcrumbs",
   "bootstrap/pagination",
   "bootstrap/pager",
   "bootstrap/labels",
   "bootstrap/badges",
   "bootstrap/jumbotron",
   "bootstrap/thumbnails",
   "bootstrap/alerts",
   "bootstrap/progress-bars",
   "bootstrap/media",
   "bootstrap/list-group",
   "bootstrap/panels",
   "bootstrap/responsive-embed",
   "bootstrap/wells",

   // Utility classes
   "bootstrap/utilities",
   "bootstrap/responsive-utilities";
}
